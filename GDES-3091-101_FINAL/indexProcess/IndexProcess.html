Index Process

Coming up with an idea

I always liked the asthetic of a terminal window and a text editor screen with code. It looked smart and interesting.

When looking for index website ideas, I thought about a website that worked as a terminal. I saw a few websites that used this concept, so by no means is this not an uncommon idea. I this really wanted to go for something that still had a terminal aspect. I decided to try to focus on a terminal theme and try to make it as interesting as I can.

I came across a website that had windows with content that you can move around here, so from there I got the idea to include windows with content. You have to type a command to open it.

While looking for more ideas, I saw some ASCII art and thought it would be cool to try to include that. I found a few websites that have ASCII art and could generate words into ASCII words.

With all this, I got my idea for the Index

Creating the Index Page

First I experimented and tested with different aspects that I wanted to include in my index on CodePen. I made movable windows with Jquery, experimented with ASCII animations and tried making some sort of background to make the page more interesting.

For the windows, I found an open source window manager here while searching how to resize and delete the Jquery windows I made. This made things alot easier for me since the windows become easily cutomizable.

Making the terminal itself was pretty tricky and very time consuming. It took a while to try to figure out how to create a new paragraph element, but I was able to find a few references of terminals on CodePen and figured it out, tho it was pretty complicated. I also was planning to use if statements to get responses, but after testing it out I figured that using switch cases would be more effiecent and easier. I made a command js and listed variables, the multi line dailogue, and ASCII art there. I added the css colours after and had fun experimenting with different colour palletes using this website. Its a pretty cool website that has a bunch of terminal colour palletes and shows the palletes in action. Im pretty glad I came across it.

I then had to think of a loading page. From the start, I thought of having and ASCII animation as my loading screen. But then thought agaisnt it after I when back here to just see it again, and it has text as when its loading, just and a console would. So I thought Id be be funny with it and got a couple of funny texts that after it all types on screen, I have a setTimeOut function wait and the use th eclear function i made to clear it and then load in the banner.

In the end, I tried adding some animated ASCII art in some way since I didnt want it to be the loading screen. Before wheni was testing things out on CodePen, I put the ASCII animations in Visual Studio Code, and for some reason the Kaomoji animation was all weird. I tesed it out with the Idex to see if i could have it above my banner cuase why not... and it worked perfectly. Im not gonna question it.

And thats about it. Took me a while to get all this done so Im glad to have finished this and actually be satisfied with it. Im definently gonna use to to make another interesting terminal website for gags or even for a game dev documentation or a portfolio in general.
